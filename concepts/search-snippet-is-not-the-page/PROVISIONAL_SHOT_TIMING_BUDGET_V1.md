# Provisional Shot Timing Budget V1 — The Search Snippet Is Not the Page

## Status note
This is a planning-stage timing budget.
It is not a script lock, visual lock, greenlight note, or upload-readiness claim.

## Why this exists
`WORDCOUNT_TIMING_CHECK_V1.md` established that the current script carries about **173 words**, or roughly **57.7–61.8 seconds** of raw spoken load at the working delivery range.

That sharpened a useful question:
- if the script already occupies that much time,
- how much visual time is realistically left for each shot,
- especially for Shot 8 and optional Shot 9?

This note turns that question into a practical first budget.

## Inputs used
This budget combines:
- `SCRIPT_DRAFT_V1.md`
- `STORYBOARD_EXPANDED_V1.md`
- `NARRATION_BEAT_TIMING_V1.md`
- `WORDCOUNT_TIMING_CHECK_V1.md`
- the current late-sequence restraint / taper logic

## Budget philosophy
The budget should preserve four priorities:
1. **Shot 4 must feel like the hinge.**
2. **Shot 5 must still read as proof, not rush by.**
3. **Shot 6 needs enough room for the date to become explanatory.**
4. **The ending should taper rather than start a second center of gravity.**

So this budget intentionally protects the middle proof chain and keeps late-sequence additions small.

## Provisional shot budget
### Shot 1 — Preview looks sufficient
- **5–6s**
- enough time for the search-result card to feel trustworthy

### Shot 2 — Confidence becomes a claim
- **5–6s**
- enough time for the conclusion overlay to appear and settle

### Shot 3 — Open the page expecting confirmation
- **3–4s**
- source-page shift should register clearly, but this shot should not linger

### Shot 4 — Qualifier changes the scope
- **5–6s**
- one of the key protected beats; the narrowing phrase must land

### Shot 5 — Direct wording compare
- **7–8s**
- protected proof beat; fast enough to stay clean, long enough to read the mismatch

### Shot 6 — Update clue explains the mismatch
- **12–13s**
- this is the largest single block because it carries both the date appearance and the denser explanation line

### Shot 7 — Trust order / evidence hierarchy
- **7–8s**
- enough time for the hierarchy to read and the routine to arrive last

### Shot 8 — Pattern label
- **2–3s**
- should work as a brief inherited label beat, not a new explanatory section

### Shot 9 — Optional second-pattern flash
- **0–2s preferred**, **3s absolute outer edge**
- if present, it should behave like a flash, not a second example block
- if it needs more than this, cutting it is safer

### Shot 10 — Viewer routine
- **6–7s**
- this is one of the real spoken landing zones and should feel usable, not rushed

### Shot 11 — Closing callback
- **4–5s**
- short clean finish; enough time for recognition and stop

## What these numbers imply
### Case A — No Shot 9
Using mid-range values, a practical no-Shot-9 shape looks like roughly:
- Shot 1: 5.5s
- Shot 2: 5.5s
- Shot 3: 3.5s
- Shot 4: 5.5s
- Shot 5: 7.5s
- Shot 6: 12.5s
- Shot 7: 7.5s
- Shot 8: 2.5s
- Shot 10: 6.5s
- Shot 11: 4.5s

**Approximate total: 61.5s** before any extra pause padding.

This is a strong sign that the current script can already support a compact ending **without** Shot 9.

### Case B — Shot 9 survives as a real flash
If Shot 9 takes about **2 seconds**, the same shape becomes roughly:
- all numbers above
- plus Shot 9: 2.0s

**Approximate total: 63.5s** before any extra pause padding.

This still fits the current short-explainer target.
But it only works if Shot 9 stays truly tiny.

## Main conclusions
### 1. The current concept does not need more total time
The measured script plus this budget both suggest that the piece can already live comfortably in the intended short range.
The challenge is not adding time.
It is spending the available time where it matters.

### 2. Shot 8 can survive, but only as a short inherited beat
The budget supports Shot 8 at roughly **2–3 seconds**.
That is enough for a label beat.
It is not enough for a new argument.

### 3. Shot 9 only survives under strict timing discipline
The budget now makes the existing qualitative rule more concrete:
- **0–2s preferred**
- **3s absolute outer edge**

If Shot 9 wants more than that, it is not functioning as the supported flash anymore.

### 4. Shot 10 and Shot 11 remain the real ending landing
The budget leaves meaningful time for:
- the routine
- the closing callback

That matches the current script structure and supports the idea that the ending should land mainly through **Shot 10 + Shot 11**, not through a richer Shot 8 / Shot 9 block.

### 5. The safest ending shape remains taper, not buildup
The budget supports the same broad shape already implied by the rough strip and the transition note:
- Shot 7 anchors
- Shot 8 labels briefly
- Shot 9 flashes only if it can stay tiny
- Shot 10 hands over the habit
- Shot 11 closes the loop

## Best use of this note
If a future read-through or rough animatic pass happens, the most useful pressure test is likely:
- whether Shot 8 feels legible inside a **2–3s** slot
- whether Shot 9 can stay meaningful inside a **0–2s preferred** slot
- whether Shot 10 and Shot 11 still feel unhurried after that

## Bottom line
The current script and measured word counts support a compact shot budget that keeps the middle proof chain protected and the late sequence deliberately small.
That is direct timing support for an ending that **tapers** rather than **rebuilds momentum**.
