# Rough Prototype Review V2 — The Search Snippet Is Not the Page

## Status note
This is a review of a rough seven-frame prototype pass.
It is not a visual lock, script lock, greenlight note, or upload-readiness claim.

## Reviewed artifact
Local contact sheet generated from the rough renderer:
- `/tmp/search_snippet_rough_frames_sheet_v2.png`

Prototype source:
- `concepts/search-snippet-is-not-the-page/render_rough_frames.py`

## What changed since V1
The prototype now includes a direct wording-compare frame between the qualifier-isolation frame and the update-clue frame.

That frame needed two passes:
1. first compare-pass: usable in principle, but too compact at review scale
2. second compare-pass: larger cards, larger type, and split-line treatment so the differing phrases become the visual event

## What this rough pass was trying to test
Whether the concept survives one extra explicit proof step without collapsing into a text wall:
1. preview feels sufficient
2. qualifier changes scope
3. wording difference can be shown directly
4. update clue still explains the mismatch
5. hierarchy card still lands as a routine

## Main result
Promising, and stronger than the six-frame version.

The new compare frame adds useful evidence rather than just repeating the live-page frame.
The first compare attempt was too compact, but the revised version is materially clearer.
Splitting each card into calmer shared wording plus one bright differing phrase makes the contrast readable enough for a rough pass.

## Frame-by-frame read

### Frames 1–4
- Still perform the same jobs as in V1.
- Frame 4 remains the strongest single frame.
- The isolated qualifier still carries the core meaning shift most efficiently.

### Frame 5 — Direct wording compare
- Now useful enough to justify its existence in the sequence.
- The two-card structure makes the wording difference explicit without forcing the viewer to parse one dense paragraph.
- The revised split-line treatment is better than the first compare attempt because the viewer's eye lands directly on the changed phrases.
- This frame is still less elegant than Frame 4, but it now earns its place as a proof step.

### Frame 6 — Update clue
- Still works as an explanation frame rather than a decorative date frame.
- After the compare frame, the date feels more causal because the wording mismatch has already been established.

### Frame 7 — Evidence hierarchy
- Still serviceable.
- The routine remains compact and non-scolding.

## Main remaining weaknesses
1. Frame 5 now works, but it is still the most text-dependent frame in the rough sequence.
2. Frame 3 remains more source-like than vivid; it mainly sets up Frames 4–6 rather than standing out itself.
3. The update-clue logic still becomes stronger with narration than with visuals alone.
4. Finish quality is still unproven; the renderer validates structure better than polish.

## Interpretation
This prototype now tests the missing side-by-side step directly, and the result is encouraging.
The compare frame did not collapse into a full text wall once the differing phrases were isolated and enlarged.

Most useful current takeaway:
- the concept is strongest when each stage has one visible job:
  - preview confidence
  - qualifier shock
  - wording compare
  - update clue

## Current judgment
Promising rough prototype, now with a viable compare step.
Still clearly prototype-grade.

Current status remains:
- rough prototype only
- not visually locked
- not script-locked
- not greenlit
