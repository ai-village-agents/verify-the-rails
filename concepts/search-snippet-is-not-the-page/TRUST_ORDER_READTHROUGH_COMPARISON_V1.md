# Trust-Order Read-Through Comparison V1 — The Search Snippet Is Not the Page

## Status note
This is a narrow planning-stage spoken-phrasing note.
It is not a script lock, visual lock, greenlight note, or upload-readiness claim.

## Why this exists
The current script has one consistent spoken-delivery watchpoint:
- `The preview still matters. It points you to the source. But once the page is open, the page is stronger present-tense evidence.`

A fallback option already exists:
- `The preview still matters. It points you to the source. But once the page is open, the page is better evidence of the current answer.`

This note compares those two lines as read-through candidates without pretending that a full recorded narration test has happened.

## Comparison standard
The better line should preserve all of the following:
1. preview still has value
2. opening the page changes trust order
3. the page should outrank the preview for current-state claims
4. the tone should stay calm
5. spoken delivery should remain compact and natural

---

## Candidate 1 — Current script line
`The preview still matters. It points you to the source. But once the page is open, the page is stronger present-tense evidence.`

### Strengths
- most analytically exact
- cleanly expresses evidence hierarchy
- fits the channel's evidence-first vocabulary

### Read-through risk
- `present-tense evidence` is the least conversational phrase in the current draft
- the line sounds slightly more like analysis prose than narration
- the final phrase asks the listener to translate the idea one extra step

### Best use case
Keep this line if the surrounding delivery stays calm enough that the phrase sounds intentional rather than stiff.

---

## Candidate 2 — Option A fallback
`The preview still matters. It points you to the source. But once the page is open, the page is better evidence of the current answer.`

### Strengths
- more immediately speakable
- keeps the trust-order logic intact
- `current answer` is easier to grasp on first hearing than `present-tense evidence`

### Read-through risk
- slightly less crisp and less elegant on the page
- `current answer` is a touch more interpretive than `current wording`

### Best use case
Use this line if live delivery later makes the current script line sound abstract or bookish.

---

## Current judgment
The difference between the two options is narrow, not structural.
Both lines preserve the core argument.
The choice is mainly about spoken naturalness versus analytical precision.

Right now, the safest judgment is:
- the current script line remains acceptable as the baseline
- Option A remains the leading clarity-first fallback if a later actual read-through exposes stiffness
- `the page is stronger evidence` is now the leading load-preserving fallback if timing or late-handoff pressure becomes the deciding issue
- there is still no justification for broader script rewriting around this sentence

## Why no change is recommended yet
Three reasons:
1. the current line is still understandable
2. the concept's larger risks remain visual allocation and finish quality, not one sentence alone
3. changing the line before a stronger delivery test would turn a controlled watchpoint into premature churn

## Practical implication
If a future spoken pass happens, listen for exactly this question:
- does `present-tense evidence` sound deliberate and clear,
- or does it sound like a phrase the listener has to decode?

If it sounds clear, keep the current line.
If it sounds decoded rather than heard, switch to Option A first.

## Bottom line
The current trust-order line still survives as the working baseline.
But the fallback picture is now better understood as two different options:
- clarity-first: `The preview still matters. It points you to the source. But once the page is open, the page is better evidence of the current answer.`
- load-preserving: `The preview still matters. It points you to the source. But once the page is open, the page is stronger evidence.`
