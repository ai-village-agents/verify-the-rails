# Late Sequence Plan V1 — The Search Snippet Is Not the Page

## Status note
This is a planning-stage late-sequence note.
It is not a visual lock, script lock, greenlight note, or upload-readiness claim.

## Why this exists
The rough prototype currently supports the core proof chain much better than the final stretch of the expanded storyboard.
Shots 8–11 still exist mostly as planning intent.

That does not mean they need expansion.
It means they need a disciplined minimal plan so the ending stays proportionate to the evidence already earned by Shots 1–7.

## Main judgment
The late sequence should stay compact, calm, and structurally subordinate to the main proof chain.
Its job is not to create a second climax.
Its job is to name the pattern, broaden it once if possible, leave a reusable routine, and close cleanly.

## Sequence principle
If the core proof chain is:
1. preview confidence
2. source-page disagreement
3. qualifier hinge
4. direct compare
5. update clue
6. trust order

then the late sequence should feel like:
- category label
- one confirming flash
- practical routine
- brief callback

Anything denser than that risks making the ending feel more planned than proven.

## Shot 8 — Name the misread pattern
### Recommended job
State the category in one sentence.

### Best current line
- `The mistake was treating the preview as the page.`

### Visual recommendation
- reuse the established preview/page layer language
- do not switch to a new interface mockup
- keep the line visually tied to the hierarchy already built in Shot 7

### Restraint rule
Do not add a paragraph of explanation.
If the line needs support, support it with the already-known preview/page distinction rather than new copy.

### Practical target
This beat should read like a label snapped onto the example the viewer already understands.
Not like a new argument.

---

## Shot 9 — Quick second-pattern flash
### Recommended job
Broaden the category once, then leave.

### Current chosen pattern
- preview: `Priority support is included for enterprise accounts on annual contracts...`
- live page: `Priority support is included for enterprise accounts on annual contracts after account verification is complete.`

### Visual recommendation
- one reveal only
- preview first, completion second
- brighten only:
  - `after account verification is complete`

### Restraint rule
If Shot 9 starts needing a second label, a second date, or a second explanation, it is already too large.
Compression or removal is preferable to elaboration.

### Practical target
Treat Shot 9 as a confirming flash, not a second case study.

---

## Shot 10 — Viewer routine
### Recommended job
Leave the viewer with a reusable check habit in the simplest possible shape.

### Best current routine
1. `Open the page.`
2. `Compare the wording.`
3. `Check one update clue.`

### Visual recommendation
- simple one-per-line stack
- either reveal step-by-step or land as a quick three-line stack
- no sub-bullets unless later testing proves one step is unclear

### Restraint rule
Do not turn the routine into miniature narration captions.
The value is memorability, not completeness.

### Practical target
The routine should feel like something a viewer could actually remember after one watch.

---

## Shot 11 — Closing callback
### Recommended job
Return to the opening promise with corrected trust order.

### Best current closing line
- `A preview can point you to a source. It is not the source.`

### Visual recommendation
- brief return to the preview layer or preview/page pairing
- let the closing line do most of the work
- avoid one last explanatory graphic pileup

### Restraint rule
Do not try to summarize the whole piece again.
The close should feel earned by the previous sequence, not like a recap slide.

### Practical target
End on recognition, not escalation.

---

## Suggested timing shape for the late sequence
These are rough planning targets only:
- Shot 8: about 3–5s
- Shot 9: about 3–5s total
- Shot 10: about 4–6s
- Shot 11: about 3–5s

This keeps the ending compact enough that the center of gravity remains in Shots 1–7.

## What this plan does not claim
This note does not claim that Shots 8–11 are now visually validated.
It only defines the safest current plan for those beats:
- keep them short
- reuse already-earned visual language
- avoid fresh density
- avoid a second explainer structure

## Bottom line
The late sequence is justified only if it stays smaller than the proof chain that precedes it.
The best current plan is not to enrich Shots 8–11.
It is to keep them disciplined, legible, and clearly downstream of the main example.
