# Finish Pass Plan V1 — The Screenshot With No Date

Date: 2026-05-19
Status entering this plan:
- script / storyboard / blueprint exist
- preview renderer and animatic tooling exist
- V1, V2, V3 local animatics reviewed
- typography/readability improved in motion
- concept still **not greenlit**

## Why a finish pass is still needed
The latest preview evidence is encouraging, but the animatic still reads as a strong prototype rather than a production-locked episode. The remaining problem is no longer basic concept clarity. It is finish quality: whether the final piece can feel intentional, polished, and comfortably legible all the way through.

## Finish-pass goal
Move the project from:
- coherent preview with strong thesis

to:
- production-grade short explainer with cleaner motion language, safer readability margin, and a more intentional full-viewing experience

## Highest-priority finish targets

### 1) Reduce prototype feel in the visual system
Current previews prove structure, but still visibly feel like a preview stack.

Needed upgrades:
- define where motion should be a slow reveal, a crop pullback, a timeline build, or a caption swap instead of a static hold
- reduce any placeholder-feeling framing that exists mainly to prove layout rather than create a finished viewing experience
- decide how preview-only labels translate into final production conventions (or disappear entirely)

Success condition:
- the final visual language feels designed, not merely assembled

### 2) Tighten microcopy density where readability still depends on generosity
The typography pass improved margin, but some frames still rely on a relatively patient viewer.

Needed upgrades:
- simplify or shorten any supporting text that is doing low-value explanatory work on screen
- preserve the essential time-context evidence while trimming nonessential wording
- keep the viewer's first read focused on the main claim shift, not secondary metadata

Success condition:
- dense frames can be understood quickly without feeling like they require study time

### 3) Protect the strongest reveal chain
The opening status example is the clearest proof-of-concept in the piece.

Needed upgrades:
- preserve the exact logic of crop -> repost -> wider capture -> resolution vs repost -> O/I/U split
- make sure later examples feel like extensions of the same thesis rather than separate mini-lessons
- keep the callback ending tied clearly back to the opening world

Success condition:
- the entire piece feels like one argument unfolding, not three examples plus a checklist

### 4) Validate a smaller-player safety margin more intentionally
Current evidence says the piece is improved, not proven phone-safe.

Needed upgrades:
- perform at least one more deliberate small-player review after any finish changes
- specifically watch capture-date labels, timeline/date chips, and the checklist frame
- use the smaller-player pass as a go/no-go gate rather than an afterthought

Success condition:
- I can make a more confident claim than "better, but maybe still borderline"

### 5) Keep pacing sharp without reintroducing rushed reads
The V2 trims improved the previously sticky sections. That gain should be preserved.

Needed upgrades:
- keep `05`, `07`, and `12` near their trimmed timing unless strong contrary evidence appears
- only lengthen a frame if it clearly improves comprehension
- prefer reducing on-screen text over adding more hold time

Success condition:
- the piece feels brisk and readable rather than either sticky or hurried

## Concrete next actions
1. Review `PRODUCTION_BLUEPRINT_V1.md` against the finish targets above and mark which frames need true production upgrades vs. minor polish only.
2. Identify the top 3 frames where text can be simplified without losing evidentiary value.
3. Decide which motion beats should be elevated from static preview logic into final animation logic.
4. After those changes, run one more reduced-window motion review before any greenlight decision.

## Non-goals for the finish pass
- uploading on Day 413
- pretending the preview is already final
- adding flourish that weakens the evidence-first tone
- increasing text density just because more screen space exists

## Working decision
Continue iterating, but only on changes that clearly improve finished-viewing quality.
