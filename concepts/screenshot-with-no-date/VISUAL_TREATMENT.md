# Visual Treatment — The Screenshot With No Date

Purpose: define a visual approach that clearly exceeds the Day 412 "static card explainer" baseline.

## Design goal
The viewer should *experience* how a screenshot feels persuasive before enough context arrives to judge it correctly.

## Visual language
### A. Tight crop -> wider truth
Repeated motif:
1. show a convincing cropped image
2. let the viewer form an impression
3. widen the frame
4. reveal the missing time marker
5. reinterpret the claim

### B. Timelines as correction tools
Whenever time matters, build a simple horizontal timeline with:
- capture time
- repost time
- later update time
- current check time

### C. Observed / Inference / Unknown overlays
Use three distinct visual states:
- **Observed** — stable blue
- **Inference** — amber
- **Unknown** — muted gray

The point is not just to say these words, but to help the viewer feel the difference between seeing, assuming, and not yet knowing.

## Suggested sequence map
### Sequence 1 — Hook (0:00-0:25)
- cold open on a tightly cropped screenshot
- large quote or status line visible
- no date visible yet
- narrator states: "This screenshot is real. The conclusion people drew from it was wrong."
- slow zoom outward reveals timestamp area still hidden at the edge

### Sequence 2 — First reveal (0:25-0:45)
- frame expands
- timestamp fades in
- timeline appears beneath the screenshot
- later update appears on the right side of the timeline
- initial conclusion gets relabeled as **Inference**

### Sequence 3 — Why this keeps happening (0:45-1:30)
Use fast, varied mini-examples:
- service-status post
- dashboard period crop
- old social post resurfacing
- announcement screenshot without version context

Each example should take the same visual form: crop -> missing time clue -> wider frame -> corrected reading.

### Sequence 4 — Where to look (1:30-2:20)
Instead of listing rules on a static card, visually point to likely time markers:
- top-right update label
- browser tab date or version context
- platform timestamp
- timezone marker
- revision history / status updates

### Sequence 5 — A tiny habit (2:20-3:00)
Show the repeatable routine as motion rather than bullets:
1. pause the repost
2. locate original source
3. check capture time and timezone
4. look for what changed after
5. share with time attached

### Sequence 6 — Closing image (3:00-end)
Return to the original screenshot, now fully visible, with a completed timeline beneath it.

Final line candidate:
**"A real screenshot can still carry the wrong moment. Check the time before you share the story."**

## Motion notes
- avoid constant motion for its own sake
- use movement only when it reveals information or changes interpretation
- hold still after each major reveal to let the viewer update their understanding

## Typography / layout notes
- larger text, fewer words per frame than Day 412 videos
- reserve full-screen text for only the strongest lines
- use screen-space hierarchy so the timestamp reveal feels important, not decorative

## Audio sync notes
- each reveal should align to a short narration beat or pause
- avoid narrating every visible word on screen
- let visuals carry some of the explanation silently

## Quality test
Before production starts, confirm at least three moments where the visual reveal teaches something the narration alone would not.
