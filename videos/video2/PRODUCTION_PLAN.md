# Production Plan - Video 2

## Research / Evidence Needs

- One reproducible example URL where response differences can be captured safely.
- Timestamped captures from at least two vantage points (different network and/or region).
- Basic response evidence (for example: visible page version markers, cache headers when available, deploy window notes).
- A clear uncertainty note for anything inferred rather than directly observed.

## Script Structure

1. **Hook (0:00-0:35):** "Same link, two different answers."
2. **Reframe (0:35-1:10):** disagreement can be infrastructure timing, not immediate proof of deception.
3. **Mechanism 1 - Caching (1:10-2:25):** plain-English cache lifecycle and stale vs fresh response.
4. **Mechanism 2 - Rollout Lag (2:25-3:40):** gradual deployment path where both versions coexist briefly.
5. **Mechanism 3 - Edge/Location (3:40-4:45):** regional propagation and edge variance.
6. **Quick Viewer Check (4:45-5:45):** check time, network/location, and one corroborating source.
7. **Close (5:45-6:20):** what this explains and what it does not explain.

## Visual Approach

- Match Video 1: clean editorial graphics, restrained motion, high legibility.
- Side-by-side request paths with explicit timestamps and region labels.
- Simple "request -> edge -> origin" diagrams instead of dense infra schematics.
- On-screen certainty tags: `Observed`, `Likely`, `Unknown`.

## Narration Approach

- Calm, direct, and non-accusatory.
- Define technical terms once, then use everyday wording.
- Keep claims scoped: narrate known facts first, then clearly mark inference.
- Avoid absolutes like "always" or "never" when discussing causes.

## Deliverables Checklist

1. Final script for 5-7 minute runtime.
2. Storyboard with cache/rollout/edge scenes.
3. Narration audio and caption file.
4. Rough cut with on-screen timestamps and certainty labels.
5. Fact-check pass against captured evidence before export.
