# Production Plan - Video 4

## Goal

Produce a 5-7 minute explainer showing how date and version drift causes real-looking online claims to conflict, even when nobody fabricated a screenshot.

## Research / Evidence Needs

- one clear example of a page updated over time
- one example where status changed across timestamped updates
- one example where draft/final or proposal/final wording changed across versions
- explicit on-screen labels for `Current`, `Historical`, and `Unknown`

## Script Structure

1. **Hook (0:00-0:35):** two captures of the same source appear to disagree.
2. **Reframe (0:35-1:10):** the question is not just truth or falsity; it is time and version context.
3. **Why This Happens (1:10-2:20):** moving pages, updated dashboards, revised documents, status posts.
4. **Worked Examples (2:20-4:35):** page deadline update, status page update, draft vs final wording.
5. **Verification Routine (4:35-5:35):** how to classify claims as current, historical, or unclear.
6. **Close (5:35-6:05):** preserve the moment, but label the moment.

## Visual Approach

- side-by-side timeline cards with visible timestamps
- version badges such as `v1`, `v2`, `Draft`, `Final`
- one persistent prompt on screen: `When was this true?`
- use color carefully: cool for direct evidence, amber for caution, red only for overreach

## Narration Approach

- explain timestamps and versions in plain language
- keep separating observed page text from inference about current status
- repeat that an outdated screenshot can be authentic and still mislead

## Deliverables Checklist

1. Video brief and production plan
2. full script with exact example wording
3. narration script for TTS
4. storyboard with update timelines and version labels
5. shot timing and rough-cut timing sheets
6. rendered storyboard, animatic, narration, rough cut, and thumbnail
7. metadata file with final title, description, chapters, and tags
